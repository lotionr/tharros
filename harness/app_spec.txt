THARROS — Real-Time AI Interview Coach
Hackathon: a16z x Overshoot x Fal x Mux Video Hackathon, May 1 2026
Build window: 4 hours, solo
Targeting: Prize #1 (Realtime/streaming) + Prize #2 (Multimodal)

=== PRODUCT DESCRIPTION ===

Tharros is a real-time AI interview coaching app. The user opens the app in their browser,
clicks "Start Session," and receives live voice coaching through an earpiece while they
practice answering interview questions.

TWO parallel live pipelines run simultaneously:

1. VIDEO PIPELINE: Webcam frames are grabbed every 800ms and sent to Overshoot VLM,
   which analyzes eye contact, posture, facial expression, and visual pacing.

2. AUDIO PIPELINE: Microphone audio is captured in 5-second chunks and sent to
   fal/Wizper (Whisper v3) for real transcription. The transcript is scanned for filler
   words (um, uh, like, you know). This is REAL speech-to-text, not a VLM estimate.

Both pipelines feed into a single debounce engine. When a signal crosses threshold 3
times in 5 seconds, ElevenLabs speaks a short coaching cue in the user's ear via
WebSocket streaming TTS. The same signal is then suppressed for 15 seconds.

The entire session is recorded via Mux Direct Upload. After the session ends:
- The Mux recording is sent to fal/openrouter (Gemini 2.5 Flash) for post-session analysis
- A written coaching report card is generated and displayed on the /playback page
- The Mux player shows chapter markers at every moment ElevenLabs fired a cue

=== TECH STACK ===

- Next.js 14 (App Router), TypeScript, Tailwind CSS
- Overshoot API: canvas frame every 800ms → VLM → visual coaching signals
- fal/Wizper: 5s mic audio chunks → Whisper v3 transcription → filler word detection
- fal/openrouter: post-session Mux URL → Gemini 2.5 Flash → written report card
- ElevenLabs: Flash v2.5, WebSocket streaming TTS, unified debounce engine
- Mux: Direct Upload + @mux/mux-player-react + custom chapter marker overlay

=== API ROUTES ===

POST /api/mux-upload
- Server-side. Creates Mux Direct Upload URL.
- Uses MUX_TOKEN_ID + MUX_TOKEN_SECRET (never exposed to client)
- Returns { uploadUrl, uploadId }

GET /api/mux-asset?uploadId=xxx
- Server-side. Polls Mux for asset ready status.
- Returns { assetId, playbackId, status }

POST /api/fal-proxy
- Server-side proxy for ALL fal API calls.
- Body: { endpoint: string, input: object }
- Calls fal.subscribe(endpoint, { input }) with FAL_KEY from env
- Returns result.data as JSON
- Used for: Wizper transcription (live) + Gemini analysis (post-session)

=== OVERSHOOT VLM PROMPT ===

System: You are an expert interview coach analyzing a video frame. Return ONLY valid JSON, no other text.

User: Analyze this interview candidate's body language and return exactly this JSON:
{
  "eye_contact": { "score": <0-10>, "cue": "<coaching cue ≤8 words, or empty>", "fire": <true if score < 5> },
  "posture":     { "score": <0-10>, "cue": "<coaching cue ≤8 words, or empty>", "fire": <true if score < 5> },
  "expression":  { "score": <0-10>, "cue": "<coaching cue ≤8 words, or empty>", "fire": <true if score < 6> },
  "pacing":      { "score": <0-10>, "cue": "<coaching cue ≤8 words, or empty>", "fire": <true if score < 4 or score > 9> }
}

Note: pacing is VISUAL pacing only (rushed movements, breathless expression).
Audio pacing and filler words are handled by the separate audio pipeline.
Cue examples: "Look at the camera", "Sit up straight", "Show more energy", "Slow down"

=== FAL/WIZPER CALL (per 5-second audio chunk) ===

endpoint: "fal-ai/wizper"
input: {
  audio_url: <fal.storage.upload(audioBlob) result>,
  task: "transcribe",
  chunk_level: "word",
  version: "3"
}

Filler words to detect: "um", "uh", "like", "you know", "so", "basically", "literally"
Fire if >= 2 fillers found in the 5-second chunk.
Cue: `You said "${firstFiller}" — keep going`

=== FAL/OPENROUTER CALL (post-session, once) ===

endpoint: "openrouter/router/video"
input: {
  video_urls: ["https://stream.mux.com/{playbackId}.m3u8"],
  model: "google/gemini-2.5-flash",
  system_prompt: "You are a professional interview coach. Be specific and constructive. Return only valid JSON, no other text.",
  prompt: "Analyze this interview practice session (under 3 minutes). Return exactly this JSON:
{
  \"overall_score\": <1-10>,
  \"confidence_rating\": \"<Poor|Fair|Good|Strong>\",
  \"top_strengths\": [\"<strength 1>\", \"<strength 2>\"],
  \"top_improvements\": [\"<improvement 1>\", \"<improvement 2>\"],
  \"summary\": \"<2-3 sentence coaching summary>\"
}"
}

=== DEBOUNCE ENGINE ===

Single engine handles ALL signal keys (eye_contact, posture, expression, pacing, filler_words).
- Track per key: { fireCount, windowStart, suppressedUntil }
- Each fire: increment fireCount; reset window if > 5s since windowStart
- Trigger: fireCount >= 3 AND Date.now() > suppressedUntil
- On trigger: speak via ElevenLabs + set suppressedUntil = now + 15000 + reset count
- Side effect: push to chapterCues array with elapsed session time
